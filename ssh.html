Processus de connexion Git d'un environnement de développement local avec cPanel et GitHub

Création du compte sur cPanel

Le compte doit être créé sur cPanel et les accès nécessaires configurés ('shell' doit être activé).

Documentation de cPanel

Créer une clé SSH

Dans le cPanel, aller sur 'Accès SSH' et générer une clé. Aller dans 'gérer' et autoriser la clé.
Ensuite, télécharger la clé privée sur l'ordinateur local (seule la clé privée est nécessaire).

Documentation de cPanel

Ajouter la clé à l'ordinateur local

Copier la clé privée dans le répertoire de clés de l’ordinateur local (Ex : $ cp /c/Users/camauger/Desktop/boiteabijoux ~/.ssh). Autoriser localement la clé avec la procédure suivante :

  • démarrer l'agent d'autorisation - eval $(ssh-agent)
  • ajouter la clé - ssh-add ~/.ssh/nomdelaclé
  • l'ajout devrait être confirmé avec un message similaire - Identity added: /c/Users/.ssh/nomdelaclé
Documentation de cPanel

Git Version Control (cPanel)

Dans la section Git Version Control de cPanel, créer un répertoire à l'emplacement voulu. Dans ce répertoire, ajouter l'identité de l'utilisateur avec 'git config' (user.email "email" et user.name "name"). Ajouter aussi la clé avec la même procédure qu'en local. Dans le répertoire, créer un fichier '.cpanel.yml' et configurer le déploiement. Ajouter les fichiers avec les commandes habituelles ('git add'). Naviguer ensuite dans l'ordinateur local, choisir le répertoi

Documentation de cPanel

Git Local

Sur l'ordinateur local, naviguer à l'emplacement voulu et faire 'git clone' (avec l'adresse fournie par cPanel). Confirmer l'ajout du serveur (Are you sure you want to continue connecting (yes/no)?). Ajouter un premier fichier (ex. echo "# README" >> README.md). Ensuite faire les commandes suivantes 'git add README.md', 'git commit -m "Initial Commit"' et 'git push -u origin master'. Faire également 'git pull' pour obtenir tout le répertoire sur le cPanel.

Documentation de cPanel

GitHub

Créer un répertoire. Ajouter la clé de déploiement (Settings > Deploy keys). Cocher la case « Allow write access ». Dans le cPanel en connexion SSH, faire les commandes suivantes :

  • git remote add origin git@github.com:PROSPECTION-huguetten/boiteabijoux.git
  • git push -u origin master
